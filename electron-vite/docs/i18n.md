# i18n

## Structure

- Base schema lives in `src/renderer/i18n/locales/zh-CN.ts`.
- Other locales must satisfy `MessageSchema` so missing keys fail at type-check time.
- Locale registration lives in `src/renderer/i18n/messages.ts`.
- Runtime setup lives in `src/renderer/i18n/index.ts`.
- Locale switching in the UI goes through `setLocale()` so storage and `html[lang]` stay in sync.

## Runtime behavior

- Saved locale in `localStorage` has the highest priority.
- If nothing is saved, the app tries `navigator.languages` / `navigator.language`.
- Regional browser locales can fall back to a supported base language, for example `en-US` -> `en`.
- `document.documentElement.lang` is updated whenever the active locale changes.

## Add a new language

1. Add `src/renderer/i18n/locales/<locale>.ts`.
2. Export a message object that `satisfies MessageSchema`.
3. Register that locale in `src/renderer/i18n/messages.ts`.
4. The locale switcher options update automatically from the registered locales.
